# Capstone Milestone M5 — Ship It + Present
**CSCI 250 · Fall 2026 · Week 17 · Due Sunday Dec 19, 11:59 PM PST · part of the 20-pt Capstone**

## Where you are in the build
This is the finish line. "My Assistant" becomes a **polished, deployed, documented product** you'd show an employer. Week 17 covers LLM/GenAI evaluation and final presentations. M5 packages everything from M1–M4 into a **public demo + README + 3-minute screencast** with an honest limitations section. This is the same submission the full **100-pt project rubric** in `Final-Project-Capstone.md` is graded against — M5 is the staged path to it.

## What to do
1. **Deploy a public demo** — a **Gradio app on Hugging Face Spaces** or a clearly-documented **Colab notebook** that a grader can open and run **first try**, on a **free tier** (no OpenAI).
2. **Write the README** with:
   - a **"What you'll see when you run this"** banner,
   - setup/run steps,
   - your **evaluation scorecard** (baseline vs. final numbers from M3/M4),
   - a **Compare-models reflection**: justify your model choice across **Claude / Gemini / Ollama** with evidence,
   - an honest **Limitations** section,
   - a short **AI-Use note** (which AI tools helped you build it).
3. **Record a 3-minute screencast** walking through the app and **one thing you'd improve**.
4. **Submit:** the demo link, the code repo, the scorecard, the screencast, and the AI-Use note by **Dec 19, 11:59 PM PST**.

## Deliverables
- Public **demo link** (HF Space or Colab) that runs reproducibly.
- **GitHub repo** + README (banner, setup, scorecard, model reflection, limitations, AI-Use note).
- **3-minute screencast.**

## Tips / common mistakes
- **Test the "first try" experience.** Open your Space/notebook in a fresh session (or have a friend try) — a broken first run is the most common point loss.
- **Don't skip limitations.** Honesty reads as maturity and is explicitly rewarded in the 100-pt rubric.
- **Show your numbers.** Put the M3/M4 scorecard right in the README — "baseline vs. final" is worth real points.
- **Compare-models reflection needs evidence**, not opinions — reference results you actually saw.
- Keep the screencast to **3 minutes**: what it does, a live run, one improvement. Done beats perfect.
