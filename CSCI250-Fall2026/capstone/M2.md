# Capstone Milestone M2 — Core Build
**CSCI 250 · Fall 2026 · Week 11 · Due Sunday Nov 8, 11:59 PM PST · part of the 20-pt Capstone**

## Where you are in the build
This is **"My Assistant" v2**: the **central capability works end-to-end**. Week 11 is RAG week (retrieval, grounding, "measure don't guess"), and Week 10 gave you embeddings + vector storage (ChromaDB). For the RAG track this milestone *is* RAG over your own docs; for the other tracks it's the equivalent "main engine" of your assistant. After M2 your assistant should do its one core thing reliably, even if it isn't pretty or hardened yet.

## What to do
1. **RAG Assistant** — Chunk your corpus, embed it, store it in a vector DB (e.g., ChromaDB). Build a **retrieve → ground → answer** pipeline so answers cite or quote retrieved context. Show it answering a question that *requires* a document.
2. **Tool-Using Agent** — Implement the **reason → act → observe** loop calling **2+ real tools**. Show a transcript where the agent picks a tool, reads the result, and continues to finish a multi-step task.
3. **Multimodal App** — Wire up the media → model → **structured output** path (e.g., image → JSON fields). Show it parsing one real example correctly.
4. **Fine-Tuned Specialist** — Assemble your dataset and run the **baseline** (the un-tuned model on your task) so you have a "before" to beat. Begin a small PEFT/LoRA run or have it queued/documented.
5. **All tracks:** keep 5–10 representative inputs handy — these become your eval set in M3.

## Tips / common mistakes
- **Grounding ≠ pasting the whole corpus.** Retrieve a few relevant chunks; passing everything wastes tokens and money.
- **Chunk size matters.** Too big buries the answer; too small loses context. Try a couple and keep notes.
- **Agents: print the trace.** You can't debug a loop you can't see — log each reason/act/observe step.
- **Fine-tune track: capture the baseline first.** No "before," no story in M3.
- Start collecting your **test inputs now** — M3's evaluation harness will reuse them.
