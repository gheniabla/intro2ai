# Assignment A9 — Build a Tested Feature with Claude Code
**CSCI 250 · Fall 2026 · Week 12 · 6 points · Due Sunday Nov 15, 2026, 11:59 PM PST**

## Goal
Practice the real skill of AI-assisted development: **directing and verifying** an AI that writes code fast. You will take a small starter repo that ships with **four failing tests**, use **Claude Code** to implement the missing feature, *read the diff*, make the tests pass, add a test of your own, and then reflect honestly on where the AI helped and where it produced wrong code. The skill is not "make the AI write code" — it's owning the result.

## What you'll use
- `content/Week12_AI-Assisted-Coding-ClaudeCode/code/starter_repo/` (the `mathkit` package + `pytest`)
- `content/Week12_AI-Assisted-Coding-ClaudeCode/code/01_ai_assisted_coding.ipynb`
- **Claude Code** (`npm install -g @anthropic-ai/claude-code`, then `claude`) — agentic coding in your terminal
- `git` for a feature branch and small, reviewed commits
- The Week 12 document (Sections 1–5) as your guide

## Tasks (~2–4 hours)
1. **See the failures first.** Copy the `starter_repo` somewhere you can edit it. From inside it, run `python -m pytest -q`. The three `apply_discount` tests pass; the **four `best_price` tests FAIL** — that is your task. Capture this red output.
2. **Branch.** Create a feature branch (`git checkout -b a9-best-price`) so your work is isolated and reviewable.
3. **Plan before editing.** Start `claude` in the repo. Note that `CLAUDE.md` tells the agent how to run tests and the conventions (round to 2 dp, never negative, reuse `apply_discount`, don't edit the tests). Ask Claude Code to **propose a plan first** for implementing `best_price(price, coupons)` — do **not** let it edit yet.
4. **Implement and READ the diff.** Have Claude Code implement `best_price` in `mathkit/discounts.py`. Actually **read the proposed diff** before accepting it. Then run `python -m pytest -q` and confirm **all tests now pass (green)**. Capture this passing output.
5. **Add your own test.** Write **one new edge-case test** in `tests/test_discounts.py` that the existing four don't cover (e.g. a coupon over 100%, a duplicate-largest coupon, or a float-rounding case). Make sure it passes too.
6. **Commit small.** Commit your change and your new test on the feature branch with a clear message.
7. **Reflect (200 words).** Write a short reflection: **where AI helped** (what it got right and saved you time on) and **where it produced wrong or risky code** (a bug, a misread requirement, an unnecessary change you caught in the diff, or something you had to correct). Be specific — name the line or behavior.
8. **Stretch (optional).** Ask Claude Code to run `/review` on your diff, or have it suggest one refactor; accept it only if you can justify it, and say in your reflection whether you kept it and why.
