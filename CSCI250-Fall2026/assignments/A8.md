# Assignment A8 — Embeddings + Semantic Search + a Simple LLM App
**CSCI 250 · Fall 2026 · Week 10 · 6 points · Due Sunday Nov 1, 2026, 11:59 PM PST**

## Goal
See *why* embeddings exist by first watching keyword search fail, then build **semantic search from scratch** — embeddings, cosine similarity by hand, and top-k retrieval — and wire it into a tiny **"ask-your-docs" LLM app**. This is the retrieval skeleton your Capstone RAG assistant grows from, so build the idea before reaching for any database.

## What you'll use
- `content/Week10_GenAI-Frameworks-Vector/code/00_embeddings_intuition.ipynb` (start here)
- `content/Week10_GenAI-Frameworks-Vector/code/02_embeddings_chromadb.ipynb`
- `content/Week10_GenAI-Frameworks-Vector/code/01_langchain_llamaindex.ipynb`
- Hugging Face `sentence-transformers` (`all-MiniLM-L6-v2`, free, local — **no OpenAI**)
- **Claude** (`claude-sonnet-4-6`) **or Gemini** (`gemini-2.5-flash`) for the generation step

## Tasks (~2–4 hours)
1. **Watch the baseline fail.** In `00_embeddings_intuition.ipynb`, run the `keyword_search` cell and confirm a meaning-based query (e.g. *"a young feline resting"*) returns nothing on a corpus that clearly contains a match. One sentence: why does it miss?
2. **Embed your own corpus.** Replace the demo corpus with **8–12 short sentences of your own** spanning 3–4 themes. Encode them with `all-MiniLM-L6-v2`.
3. **Cosine similarity by hand.** Implement `cosine(a, b)` with NumPy, build the pairwise similarity matrix, and draw it as a **heatmap**. Point to one bright block and name the theme the model found with no labels.
4. **Top-k search from scratch.** Write `semantic_search(query, ...)` that cosine-compares a query to every vector and returns the top-3. Run it on a query whose words do **not** literally appear in the matched sentence, and show it succeeds where keyword search failed.
5. **See the scaled version.** In `02_embeddings_chromadb.ipynb`, load the **same corpus** into a ChromaDB collection and run `collection.query(...)`. Confirm the top hit matches your hand-built result, and note the distance-vs-similarity flip (smaller distance = larger similarity).
6. **A simple LLM app.** Build a tiny "ask-your-docs" cell: retrieve the top-3 chunks for a question with your search, stuff them into a prompt as context, and generate a grounded answer with **Claude or Gemini**. Ask it one question your corpus *can* answer and one it *can't*; note what it does in each case.
7. **Stretch (optional).** Swap the generation model (Claude ↔ Gemini) and compare the two answers in 2–3 sentences — *or* rebuild the retrieval step with LangChain's `Chroma` + `HuggingFaceEmbeddings` wrapper from `01_langchain_llamaindex.ipynb` and confirm you get the same top hit.
