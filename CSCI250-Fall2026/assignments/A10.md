# Assignment A10 — A Tool-Using Agent with a Safety Guardrail
**CSCI 250 · Fall 2026 · Week 13 · 6 points · Due Sunday Nov 22, 2026, 11:59 PM PST**

## Goal
Build a small **tool-using agent** that runs the *reason → act → observe* loop, then make it **safe** and **measurable**. You will add a guardrail that defends against **prompt injection** smuggled in through tool output, and produce a small **eval scorecard** that scores both *did the agent pick the right tool?* and *did it give the right answer?* This is the v3 capability your Capstone agent needs.

- `content/Week13_Agents-MCP/code/03_agent_safety_and_eval.ipynb` (injection demo + scorecard — runs **without** an API key)
- `content/Week13_Agents-MCP/code/agent_app.py` (optional Flask serving)

- The Week 13 document (Sections 2–4, 7) as your guide

## Tasks (~2–4 hours)
1. **Get the loop running.** In `01_agents_tool_calling.ipynb`, run the Claude agent loop with at least **two tools**, **one of which you add yourself** (e.g. a unit converter, a word counter, or a small fake-database lookup). Keep the **`max_turns` guard** in place.
2. **Show a real tool call.** Capture a transcript where the agent *chooses* to call a tool and uses the result in its final answer (don't just call the tool yourself).
3. **Run the injection attack.** In `03_agent_safety_and_eval.ipynb`, run the prompt-injection demo where a malicious **tool result** ("IGNORE ALL PREVIOUS INSTRUCTIONS…") hijacks the naive agent. Show the naive agent obeying it.
4. **Add a guardrail.** Defend with at least **two independent layers** from the document: a least-privilege `ALLOWED_TOOLS` set (a "read-only" agent simply *cannot* call a dangerous tool), an input/output scan that flags injection patterns and re-asserts that tool output is **data not instructions**, plus the bounded `max_turns`. Show the guarded agent **refusing** the same attack.
5. **Score it.** Build a small **eval scorecard**: 5+ test cases, each with an expected **tool** and expected **answer**. Score `int(tool == expected_tool) + exact_match(answer, expected)` per case and print the scorecard. **Add one eval case for the tool you wrote** and keep the scorecard passing.
6. **Explain the design (250 words).** Write a design note: what tools you exposed and why, how your loop stays bounded and safe (least privilege, max-turns, untrusted tool output), and a 3–4 sentence explanation of **MCP** in your own words (client vs. server, why a shared protocol beats one-off integrations).
7. **Stretch (optional).** Serve your agent over a Flask `/ask` endpoint (adapt `agent_app.py`) with a working `curl`/`requests` example — *or* try a second injection payload your guardrail must also block, and add it as a regression case in your scorecard.
